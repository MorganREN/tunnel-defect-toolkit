# Artifact Schema

This document describes the main V1 output artifacts.

## `manifest.csv`

Generated by:

```bash
tdt manifest CONFIG --out manifest.csv
```

Columns:

- `schema_version`: currently `tdt-manifest-v1`
- `image_id`
- `image_path`
- `mask_path`
- `split`

## `splits.csv`

Generated by:

```bash
tdt split CONFIG --out splits/
```

Columns:

- `schema_version`: currently `tdt-splits-v1`
- `image_id`
- `image_path`
- `mask_path`
- `split`

## `tile_manifest.csv`

Generated by:

```bash
tdt tile CONFIG --out tiles/
```

Columns:

- `schema_version`: currently `tdt-tile-manifest-v1`
- `tile_id`
- `source_id`
- `split`
- `image_path`
- `mask_path`
- `x0`, `y0`, `x1`, `y1`
- `tile_width`, `tile_height`
- `source_width`, `source_height`
- `tile_size`
- `stride`

## `morphology_descriptors.csv`

Generated when `tdt analyze --with-morphology` is used.

Each row is one connected region in a semantic mask. Morphometric size columns
are expressed in pixels.

Columns:

- `class_id`
- `instance_id`
- `area_px`
- `perimeter_px`
- `compactness`
- `elongation`
- `eccentricity`
- `solidity`
- `bbox_width_px`
- `bbox_height_px`
- `skeleton_length_px`
- `mean_width_px`
- `orientation`
- `image_id`
- `class_name`

Same-class regions touching under the configured `connectivity` are recorded as
one region. Regions smaller than `min_area_px` are omitted.

## `analysis_metadata.json`

Generated by every `tdt analyze` run.

Fields include:

- `schema_version`: currently `tdt-analysis-metadata-v1`
- `tool`: package name and version
- `dataset`: dataset identity, annotation format, class mapping, and ignored id
- `input_manifest`: image/mask counts and deterministic manifest SHA-256 digest
- `quality_policy`: small-foreground threshold
- `morphology_policy`: enabled state, pixel units, connectivity, area threshold,
  and same-class touching-region interpretation
- `run`: UTC start time and duration
